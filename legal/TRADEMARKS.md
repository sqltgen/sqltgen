# sqltgen Trademark and Brand Policy

`sqltgen`, the sqltgen logo, `sqltgen.org`, and official project branding are
trademarks/brand assets controlled by the project maintainers.

## What is allowed

- Referencing the project by name for factual purposes (for example,
  "compatible with sqltgen").
- Linking to the official project pages.
- Describing your fork or distribution as "based on sqltgen".

## What is not allowed without prior written permission

- Using `sqltgen` as the name of a forked product in a way that causes likely
  confusion about source, affiliation, sponsorship, or endorsement.
- Using official logos or visual identity for non-official releases.
- Registering domains, package names, or organization names that are likely to
  be mistaken for official sqltgen properties.

## Fork naming guidance

If you publish a fork, use a distinct project name and include a clear note
such as: "This is an independent fork of sqltgen and is not affiliated with
the official sqltgen project at sqltgen.org."

## Scope

This policy governs trademark and brand usage only. Copyright licensing for
source code is governed by the license files in this folder.
